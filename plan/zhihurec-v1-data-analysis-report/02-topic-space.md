# Subproblem 2: Topic Space

## 1. Goal
Complete report section 3 by showing topic volume, topic exposure concentration,
and a compact topic co-occurrence view.

## 2. Why this step exists
The system uses topic IDs for recall, reranking, query-topic mapping, and cold-start
profile blending. The report needs to show that topics form a useful, structured
space rather than arbitrary IDs.

## 3. Files involved
- `scripts/eda.py` - extend the script with topic analysis functions.
- `docs/data_analysis_report.md` - fill section 3.
- `docs/figs/` - add topic figures.
- `data/zhihurec_1m/raw/info_answer.csv` - read-only source for answer topic IDs.
- `data/zhihurec_1m/raw/info_question.csv` - read-only source for question topic IDs.
- `data/zhihurec_1m/raw/inter_impression.csv` - read-only source for answer exposure weighting.
- `data/zhihurec_1m/raw/info_topic.csv` - read-only source for total topic count.

## 4. Exact changes
- Add topic-count and topic-exposure calculations to `scripts/eda.py`.
- Weight answer topics by answer impression counts to estimate topic exposure.
- Compute top-100 topic exposure share.
- Build a top-50 topic co-occurrence graph from answer and question topic lists.
- Write `docs/figs/05_topic_exposure_top20.png`.
- Write `docs/figs/06_topic_exposure_cumulative.png`.
- Write `docs/figs/07_topic_cooccurrence_top50.png`.
- Update section 3 in `docs/data_analysis_report.md` with conclusions.

## 5. Out of scope
- Do not resolve topic IDs into names; the dataset only provides IDs.
- Do not use network graph libraries unless already available; matplotlib is enough.
- Do not change backend ranking code.

## 6. Done condition
This step is done when section 3 contains real topic metrics and three topic
figures generated by `scripts/eda.py`.

## 7. Verification
Run:


Expected result: all three figures exist and section 3 mentions the top-100 topic
exposure share.

## 8. Expected output
- Three topic figures under `docs/figs/`.
- Section 3 completed in `docs/data_analysis_report.md`.

## 9. Notes for the next step
Step 3 can reuse topic exposure and topic parsing helpers to compute query-topic
hit counts.

## 10. Risks or ambiguity
The co-occurrence graph is an explanatory visualization, not a learned taxonomy.
The report should present it as evidence that topic IDs cluster through shared
answers/questions, not as semantic labels.
