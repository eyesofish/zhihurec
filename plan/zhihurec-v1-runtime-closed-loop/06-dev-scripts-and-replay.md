# Subproblem 6: Dev Scripts And Replay

## 1. Goal
Add developer scripts that make the local demo repeatable:

- initialize MySQL from schema and seed SQL
- reset the demo user profile
- replay a small event sequence through the HTTP API

## 2. Why this step exists
The brief requires local single-machine operation and scriptable verification. The project should not depend only on hand-clicking the frontend.

The current repository already has build and import-pack scripts, but it does not yet have runtime initialization or replay scripts.

## 3. Files involved
- `scripts/import_demo_world.py` - already regenerates `build/demo_world/import_demo_world.sql`; keep it as the seed SQL generator.
- `scripts/apply_demo_mysql.py` - new script that applies `sql/v1_schema.sql` and `build/demo_world/import_demo_world.sql` using `ZHIHUREC_DATABASE_URL`.
- `scripts/reset_demo_user.py` - new script that resets the demo user's `user_profile` from `build/demo_world/demo_user_profile_seed.json`.
- `scripts/replay_demo_events.py` - new script that reads `build/demo_world/demo_event_replay.jsonl` and calls the backend HTTP endpoints.
- `backend/README.md` - document script usage.
- `docs/v1_local_runbook.md` - optional new runbook if the README becomes too long.

## 4. Exact changes
- Create `scripts/apply_demo_mysql.py`.
- The script should:
  - read `ZHIHUREC_DATABASE_URL`
  - accept `--schema-sql`, default `sql/v1_schema.sql`
  - accept `--seed-sql`, default `build/demo_world/import_demo_world.sql`
  - accept `--dry-run`
  - in dry-run mode, verify both files exist and print the target database from the URL
  - in normal mode, connect with PyMySQL and apply both SQL files in order
- Create `scripts/reset_demo_user.py`.
- The script should:
  - read `ZHIHUREC_DATABASE_URL`
  - accept `--profile-seed`, default `build/demo_world/demo_user_profile_seed.json`
  - update only the one demo user's `user_profile`
  - not delete content tables
- Create `scripts/replay_demo_events.py`.
- The script should:
  - accept `--base-url`, default `http://127.0.0.1:8000`
  - accept `--input`, default `build/demo_world/demo_event_replay.jsonl`
  - accept `--limit`, default a small number such as 20
  - send `search_query` events through `POST /search`
  - send `recommendation_click` events through `POST /event/recommendation_click`
  - send `search_result_click` events through `POST /event/search_result_click`
  - print each request type and whether it succeeded
- Update `backend/README.md` with the local order:
  1. install backend requirements
  2. regenerate import SQL
  3. set `ZHIHUREC_DATABASE_URL`
  4. run `scripts/apply_demo_mysql.py`
  5. start backend
  6. run smoke requests
  7. optionally run replay

## 5. Out of scope
- Do not download raw data.
- Do not rebuild `build/demo_world/` unless the user explicitly asks.
- Do not require Docker.
- Do not require the `mysql` command-line client because it is not currently available on PATH.
- Do not start background services automatically in this step.

## 6. Done condition
The scripts can be dry-run without a database.

With a valid MySQL URL, they can initialize the schema/data, reset the demo profile, and replay a short event sequence.

## 7. Verification
Dry-run checks:

```powershell
& 'C:\ProgramData\anaconda3\python.exe' scripts\import_demo_world.py --input-dir build\demo_world --output-sql build\demo_world\import_demo_world.sql --truncate-first
& 'C:\ProgramData\anaconda3\python.exe' scripts\apply_demo_mysql.py --dry-run
```

Expected result:

- import SQL generation reports `44690 row(s) across 12 table payload(s)`
- dry-run reports the schema and seed SQL files exist

With MySQL configured:

```powershell
$env:ZHIHUREC_DATABASE_URL='mysql+pymysql://user:password@localhost:3306/zhihurec_demo'
& 'C:\ProgramData\anaconda3\python.exe' scripts\apply_demo_mysql.py
& 'C:\ProgramData\anaconda3\python.exe' scripts\reset_demo_user.py
```

Expected result:

- schema and seed data are applied
- demo profile is reset without changing content tables

With backend running:

```powershell
& 'C:\ProgramData\anaconda3\python.exe' scripts\replay_demo_events.py --limit 10
```

Expected result:

- script prints successful calls for search and click events
- `/debug/profile` changes after replay

## 8. Expected output
Runtime developer scripts and updated backend run instructions.

## 9. Notes for the next step
Step 7 can point the frontend runbook to these scripts instead of duplicating setup instructions.

## 10. Risks or ambiguity
Applying multi-statement SQL through PyMySQL needs careful statement splitting. Keep the SQL files generated by our own scripts simple, and add clear errors if a statement fails.
