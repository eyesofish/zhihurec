# 子问题 2：本地目录与产物落地

## 1. 目标
创建用户要求的本地目录结构；如果官方来源允许继续下载，则把数据文件、校验报告和检查脚本都保存到指定位置。

## 2. 为什么需要这一步
用户希望当前工作区内有固定且可复现的目录结构，还希望保留可检查的结果文件，以及一个能够解释字段含义和主外键关系的脚本。

## 3. 涉及文件
- `data/zhihurec_1m/raw` - 数据文件的目标目录。
- `data/zhihurec_1m/meta` - 校验结果和元数据输出目录。
- `scripts` - 辅助脚本目录。
- `data/zhihurec_1m/meta/check.txt` - 用于保存文件完整性、非空检查、样例行和行数统计。
- `scripts/inspect_zhihurec.py` - 读取 8 个文件，输出字段说明和主外键关联检查。

## 4. 具体改动
- 创建 3 个目标目录；如果目录已存在则直接复用。
- 如果允许自动下载，则只把官方 1M 相关的文件放入 `data/zhihurec_1m/raw`。
- 如果官方来源只提供 8 个基础文件，而 README 明确说明 1M 需要通过“取前 N 行”生成，就记录并按该规则生成 1M，不默认下载 100M 全量。
- 对每个 CSV 文件检查是否存在、是否非空，打印前 3 行，并统计总行数，将结果写入 `data/zhihurec_1m/meta/check.txt`。
- 创建 `scripts/inspect_zhihurec.py`，让它读取 8 个文件、打印 README 中的字段说明、抽样检查键关联，并总结哪些表可以互相关联。

## 5. 不在本步骤范围内的内容
- 默认下载 100M 全量数据。
- 下载无关论文、代码或其他镜像。
- 构建后续推荐模型或分析流水线。

## 6. 完成条件
目标目录全部存在；如果允许下载，则 8 个数据文件已到位并完成检查，同时 `check.txt` 和 `inspect_zhihurec.py` 都已生成。

## 7. 验证方式
- 运行目录列表命令，确认目标目录存在。
- 如果已下载文件，则运行文件大小、行数和样例行检查。
- 下载后运行 `python scripts/inspect_zhihurec.py`，确认脚本能打开这些文件并打印预期摘要。

## 8. 预期输出
本地数据目录、校验报告，以及一个可重复使用的检查脚本。

## 9. 给下一步的说明
如果第 1 步仍然被人工确认阻塞，那么当前只完成目录创建；如果已经确认可继续，则本步骤可以继续完成下载、裁切和校验。

## 10. 风险或歧义
官方来源可能不会提供单独的 1M 压缩包。在这种情况下，必须按照 README 的 top-N 规则在本地生成 1M，而不是自行发明采样方法。
