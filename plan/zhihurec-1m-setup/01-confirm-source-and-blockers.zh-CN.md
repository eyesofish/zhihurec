# 子问题 1：确认官方来源与阻塞点

## 1. 目标
核对 THUIR 官方 `ZhihuRec-Dataset` README 中的关键说明，并判断官方下载路径是否可以直接在终端中使用，还是必须先经过人工确认。

## 2. 为什么需要这一步
用户明确要求只能使用官方来源，不要误下 100M 全量数据；同时如果官方入口只是网页下载页，就必须停下来说明情况，而不是默认继续。

## 3. 涉及文件
- `plan/zhihurec-1m-setup/README.md` - 用来跟踪任务状态和步骤依赖。
- `plan/zhihurec-1m-setup/01-confirm-source-and-blockers.md` - 用来记录本步骤的确认内容。

## 4. 具体改动
- 打开官方 `THUIR/ZhihuRec-Dataset` README。
- 确认 `ZhihuRec-1M` 被描述为官方提供的小版本。
- 确认 README 中的数据集说明列出了 8 个文件。
- 确认 README 的下载入口指向官方数据下载链接。
- 检查该官方链接，判断它是文件直链还是共享网页。
- 如果它是共享网页，就在自动下载之前停下，并明确报告阻塞点。

## 5. 不在本步骤范围内的内容
- 下载任何数据文件。
- 创建本地检查脚本。
- 写入数据检查结果文件。

## 6. 完成条件
对 README 中每一项用户要求都给出明确的“是/否”结论，并清楚说明终端下载能否继续，还是必须先人工确认。

## 7. 验证方式
- 打开官方 README，并引用对应内容。
- 检查官方下载链接的 HTTP 响应或页面内容。
- 确认该链接最终指向的是目录共享网页还是文件直链。

## 8. 预期输出
得到一个有依据的结论，说明后续流程能否自动继续。

## 9. 给下一步的说明
如果官方链接是共享网页，第 2 步应该先停在安全的本地目录创建阶段。如果用户之后确认可以继续，第 2 步再下载准确的官方文件。

## 10. 风险或歧义
共享页面可能通过 JavaScript API 暴露文件，而不是普通链接。即便技术上能够直接下载，用户要求也明确说明：如果官方入口是网页下载页，就必须先停下并说明情况。
